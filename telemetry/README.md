# aLog Telemetry Stack

Стек сбора, хранения и визуализации метрик в реальном времени для проекта **aLog**.

## 🚀 Компоненты стека

- **InfluxDB 2.7** — высокопроизводительная база данных временных рядов для хранения телеметрии.
- **Telegraf 1.30** — агент сбора данных, принимающий UDP-пакеты от ESP32 на порту `5000`.
- **Grafana** — платформа визуализации с преднастроенным дашбордом мониторинга телеметрии.

## 📂 Структура каталогов

```text
telemetry
├── data/
│   ├── grafana/                      # Локальное состояние Grafana: база, кеши и плагины (Ignored in Git)
│   └── influxdb/                     # Локальное состояние InfluxDB: данные, WAL и метаданные
├── grafana/                          # Настройки и автопровижининг Grafana
│   ├── provisioning/                 # Автоматическая загрузка источников данных и дашбордов при старте
│   │   ├── dashboards/               # Провижининг дашбордов
│   │   │   ├── alog_dashboard.json   # JSON-модель панели мониторинга aLog
│   │   │   └── dashboards.yaml       # Конфигурация провайдера дашбордов
│   │   └── datasources/              # Провижининг источников данных
│   │       └── influxdb.yaml         # Подключение InfluxDB (протокол Flux, токен доступа)
│   └── grafana.ini                   # Основные настройки Grafana
├── .env                              # Локальные секреты и переменные окружения (не коммитится)
├── .env.example                      # Шаблон переменных окружения для быстрой настройки
├── .gitignore                        # Исключение локальных данных (data/) и секретов (.env) из Git
├── docker-compose.yml                # Оркестрация контейнеров InfluxDB, Telegraf и Grafana
├── README.md                         # <--- Этот файл
├── run.sh                            # Скрипт запуска и сброса локального состояния
└── telegraf.conf                     # Конфигурация парсера CSV и пересылки UDP-пакетов в InfluxDB
```
## 🛠️ Описание ключевых файлов и директорий

### `docker-compose.yml`
- Все постоянные данные сохраняются **локально в папке `./data/`** проекта (bind mounts), а не в глобальных именованных томах Docker.
- Жёстко фиксирует порядок запуска (`depends_on: influxdb`).

### `telegraf.conf`
- **Вход (Input):** Слушает UDP-сокет `udp://:5000`, на который микроконтроллер ESP32 отправляет телеметрию.
- **Парсер:** Использует формат `csv` с менами колонок `timestamp, voltage, voltage_err, current, current_err, power, relay1, relay2`.
- **Выход (Output):** Записывает метрики в InfluxDB в measurement `power_monitor`.

### `grafana/` (Автопровижининг)
- `grafana.ini` разрешает анонимный вход (`[auth.anonymous] enabled = true` с ролью `Viewer`).
- `provisioning/` автоматически подключает InfluxDB и загружает дашборд `alog_dashboard.json` при каждом запуске контейнера

## Быстрый старт

### 1. Настройка переменных окружения
```bash
cp .env.example .env
# Отредактировать .env и установить пароли и токены
nano .env
```

### 2. Запуск стека в один клик
Запустить скрипт run.sh (создаст директории и поднимет контейнеры):
```bash
./run.sh
```
*(Альтернативно: `docker compose up -d`)*

### 3. Переносимость и повторяемый запуск
- `grafana/provisioning` и `telegraf.conf` — источник истины для визуализации и сбора метрик.
- `data/grafana` — хранит runtime state Grafana. Не является переносимым конфигом.
- Для смены provisioning-файлов необходимо очистить состояние Grafana:
  ```bash
  ./run.sh reset-grafana
  ```
- Если нужно начать с чистого листа для всего стека:
  ```bash
  ./run.sh reset-all
  ```

### 4. Как развернуть на новом компьютере
1. Клонировать репозиторий.
2. Скопировать шаблон переменных окружения:
   ```bash
   cp .env.example .env
   ```
3. Отредактировать `.env` и указать свои секреты.
4. Запустить стек:
   ```bash
   ./run.sh
   ```

### Доступ к сервисам

|           Сервис            |          URL          |              Учётные данные              |
| :-------------------------- | :-------------------- | :--------------------------------------- |
| **Grafana** (Дашборды)      | http://localhost:3000 | Анонимный вход (или `admin` / `admin`)   |
| **InfluxDB** (БД и запросы) | http://localhost:8086 | `admin` / пароль из `$INFLUXDB_PASSWORD` |
| **Telegraf** (Прием данных) | UDP порт `5000`       | —                                        |

## 📡 Поток данных и формат пакетов

### Архитектура передачи
```text
[ ESP32-C3 / aLog ] ---( UDP пакеты на порт 5000 )---> [ Telegraf ] ---( Протокол Flux )---> [ InfluxDB ] ---> [ Grafana ]
```

### Структура CSV-пакета от ESP32
Микроконтроллер отправляет строку в формате CSV без заголовка, разделённую точкой с запятой:
```text
1686300000.0;230.5;0;5.2;1;1151.6;1;0
```

| Колонка |  Имя поля     |   Тип    |                                   Описание                                    |
| :-----: | :------------ | :------: | :---------------------------------------------------------------------------- |
| 1       | `timestamp`   | `string` | Временная метка (игнорируется Telegraf, используется текущее время получения) |
| 2       | `voltage`     | `float`  | Измеренное напряжение сети в Вольтах (В)                                      |
| 3       | `voltage_err` | `int`    | Флаг ошибки АЦП при измерении напряжения (`0` — ок, `1` — ошибка)             |
| 4       | `current`     | `float`  | Измеренный ток нагрузки в Амперах (А)                                         |
| 5       | `current_err` | `int`    | Флаг ошибки АЦП при измерении тока (`0` — ок, `1` — ошибка)                   |
| 6       | `power`       | `float`  | Расчетная активная мощность в Ваттах (Вт)                                     |
| 7       | `relay1`      | `int`    | Состояние реле канала 1 (`0` — выключено, `1` — включено)                     |
| 8       | `relay2`      | `int`    | Состояние реле канала 2 (`0` — выключено, `1` — включено)                     |

## 🔧 Обслуживание и управление

### Просмотр журналов в реальном времени
```bash
docker compose logs -f telegraf
docker compose logs -f influxdb
docker compose logs -f grafana
```

### Остановка и удаление

**Простая остановка** (сохраняет контейнеры, настройки и все накопленные данные):
   ```bash
   docker compose stop
   ```
   *(Повторный запуск: `docker compose start`)*

**Остановка с удалением контейнеров** (удаляет временные контейнеры и внутренние Docker-объемы, `data/` сохраняется):
   ```bash
   docker compose down -v
   ```

**Полный сброс данных** (останавливает контейнеры и удаляет `data/` с базой и дашбордами):
   ```bash
   docker compose down -v && sudo rm -rf data/
   ```

**Полная очистка системы** (удаляет контейнеры, `data/` и Docker-образы):
   ```bash
   docker compose down -v --rmi all && sudo rm -rf data/
   ```
