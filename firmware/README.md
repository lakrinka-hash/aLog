# aLog Firmware

Исходный код прошивки для микроконтроллера ESP32-C3 проекта **aLog** — системы автономного логирования и телеметрии.

## Framework

Espressif IoT Development Framework (**ESP-IDF**) v6.0.1

## 🛠️ Описание ключевых директорий

### `.vscode/`
Содержит настройки рабочей области Visual Studio Code для текущего локального окружения разработки.
* **Локальный тулчейн:** Конфигурация адаптирована под локальную систему пользователя (`/home/termit/`) и использует установленный тулчейн `riscv32-esp-elf-gcc` версии 15.2.0 из состава ESP-IDF v6.0.1.
* **Интеграция с CMake:** Файл `c_cpp_properties.json` связывает IDE с автоматически генерируемым файлом `compile_commands.json` в каталоге сборки.
* **Аппаратная отладка и прошивка:** Файлы `launch.json` и `settings.json` настраивают работу со встроенным в чип ESP32-C3 USB-JTAG отладчиком (конфигурация `esp32c3-builtin.cfg`).
* **Статический анализ кода:** Интеграция `esp-clang` и флаг `--clang-tidy` в `settings.json` активирует строгий аудит кода «на лету» в среде разработки.

### `components/`
Здесь собраны независимые, слабо связанные модули:
* `sd_spi_drv` — Низкоуровневый драйвер SD-карты по шине SPI с интеграцией в виртуальную файловую систему ESP-IDF (VFS FATFS). Обеспечивает монтирование, управление выводом CS и проверку состояния карты.
* `ads1115_drv` — Драйвер внешнего 16-битного АЦП ADS1115. Обеспечивает настройку скорости выборки (SPS), программируемого усилителя (PGA) и чтение каналов измерения напряжения и тока.
* `ssd1306_drv` — Драйвер монохромного OLED-дисплея SSD1306. Адаптирован под I2C Master API (`driver/i2c_master.h`) из ESP-IDF v5.3+ / v6.0+.
* `gfx` — Высокоуровневый графический движок. Работает через локальный фрейм-буфер в DRAM. Предоставляет функции вывода текста и геометрических примитивов. 
* `mono_fonts` — Библиотека моноширинных растровых шрифтов для графического модуля `gfx`
* `wifi_app` — Модуль инициализации Wi-Fi в режиме клиента (Station). Обрабатывает системные события подключения, получения IP-адреса и реализует логику автоматического переподключения.

### `main/`
Главный модуль приложения, реализующий бизнес-логику и координацию всех подсистем:
* `main.c` — Точка входа `app_main()`. Инициализирует шины I2C и SPI, монтирует SD-карту, запускает сетевые службы и инициализирует все задачи FreeRTOS.
* `main.h` — Список резервных конфигурационных констант, используемых при автономной сборке отдельных модулей вне среды Kconfig.
* `system_state.c/h` — Централизованное хранилище глобального состояния системы (данные АЦП, состояние реле, статус SD и Wi-Fi). Доступ к данным защищен мьютексом FreeRTOS для потокобезопасного обмена между задачами.
* `system_storage.c/h` — Высокоуровневый менеджер записи данных. Отвечает за буферизованную запись логов на SD-карту, периодический сброс буферов (`fflush`/`fsync`) и ротацию лог-файлов.
* `system_time.c/h` — Модуль синхронизации системного времени через протокол SNTP с автоматической установкой часового пояса.
* `web_server.c/h` — Встроенный HTTP-сервер на базе `esp_http_server` для удаленного мониторинга телеметрии и управления состоянием реле через веб-браузер в локальной сети.
* `sysyem_tasks/` — Пакет независимых асинхронных задач FreeRTOS:
  * `adc_task` — задача периодического чтения показаний АЦП и обновления системного состояния.
  * `lcd_task` — задача формирования графического кадра и вывода телеметрии на экран SSD1306.
  * `log_task` — задача периодического сохранения измерений на SD-карту.
  * `pwr_task` — задача контроля питания, реагирующая на сбои для безопасного завершения работы с файловой системой.
  * `udp_task` — задача трансляции телеметрических данных по протоколу UDP на локальный сервер (Telegraf/InfluxDB).
  * `wifi_task` — задача мониторинга сетевого подключения.
  * `tasks_list.h` — глобальный реестр хэндлов задач (`system_task_t`), позволяющий управлять жизненным циклом потоков.

### Корневые файлы конфигурации
* `.clangd` — Конфигурация языкового сервера Clangd. Принудительно удаляет специфичные для GCC флаги оптимизации (`-fno-shrink-wrap`, `-fno-malloc-dce` и др.) из базы компиляции для корректной работы линтера в IDE.
* `CMakeLists.txt` —  Главный сценарий системы сборки CMake, задающий минимальную версию и регистрирующий проект в экосистеме ESP-IDF.
* `Kconfig.projbuild` — Файл определения параметров меню конфигурации (`idf.py menuconfig`). Позволяет без правки кода настраивать GPIO пины, I2C адреса, параметры АЦП, размеры стеков FreeRTOS задач, а также Wi-Fi SSID, пароль и IP-адрес сервера телеметрии.
* `idf_component.yml` — Манифест внешних зависимостей. Указывает менеджеру компонентов ESP-IDF необходимость загрузки пакета `espressif/mdns` из официального реестра.
* `sdkconfig.defaults` — Заводские настройки сборки по умолчанию: выбор чипа ESP32-C3 (`CONFIG_IDF_TARGET="esp32c3"`), объем флеш-памяти 4MB, тактовая частота процессора 160 МГц, оптимизация размера бинарного файла и частота тиков FreeRTOS 1000 Гц.
